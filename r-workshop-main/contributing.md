# Contributing to R Workshop

First off, thank you for considering contributing to this project! Your help is greatly appreciated.

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please open an issue and provide the following information:

- A clear and descriptive title for the issue.
- A step-by-step description of how to reproduce the bug.
- The expected behavior and the actual behavior.
- Your R version (`R.version.string`), operating system, and the versions of any relevant packages.

### Suggesting Enhancements

If you have an idea for an enhancement, please open an issue and provide the following information:

- A clear and descriptive title for the issue.
- A detailed description of the proposed enhancement and why it would be beneficial.

### Pull Requests

If you would like to contribute code, please follow these steps:

1. Fork this repository.
2. Create a branch: `git checkout -b <branch_name>`.
3. Make your changes and commit them with a descriptive commit message: `git commit -m '<commit_message>'`.
4. Push to your forked repository: `git push origin <branch_name>`.
5. Open a pull request to the `main` branch of this repository.

## Styleguides

### R Code

Please follow the [Tidyverse style guide](https://style.tidyverse.org/) for R code. You can use the `styler` package to automatically format your code.

### Commit Messages

Please write clear and concise commit messages. The first line of the commit message should be a short summary of the changes (less than 50 characters). The body of the commit message should provide a more detailed explanation of the changes.
